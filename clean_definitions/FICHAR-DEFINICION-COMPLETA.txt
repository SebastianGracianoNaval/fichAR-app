1. VISIÓN Y CONTEXTO
================================================================================
fichAR: Sistema de Gestión y Administración de Cumplimiento Laboral para empresas y PyMEs argentinas.
Filosofía: "Código de Grado Bancario, Interfaz de Grado Humano".
Contexto 2026: Reforma Laboral, informalidad 40-45%, RIFL, FAL, ARCA.
Normativa: LCT 20.744, Art. 52, 197 bis, 198, 210, Ley 11.544.
Compatibilidad: Android API 21+, iOS 12+, Windows 10+, macOS 10.14+, Ubuntu 18.04+.

2. JERARQUÍA ORGANIZACIÓN → SUCURSAL → SUPERVISOR → EMPLEADOS
================================================================================

--- 2.1 Estructura ---
Organización (empresa)
  └── Sucursal (opcional; una org puede tener 0 o N sucursales)
        └── Supervisor (asignado a sucursal y/o empleados)
              └── Empleados (asignados a supervisor y sucursal)

--- 2.2 Reglas ---
- Org sin sucursales: branch_id nullable, todos en "raíz". Comportamiento como single-location.
- Org con sucursales: cada empleado pertenece a 0 o 1 sucursal. Lugares de trabajo pertenecen a sucursal.
- Supervisor: puede estar asignado a una sucursal (ve solo empleados de esa sucursal) o a empleados específicos (independiente de sucursal).
- Un empleado tiene 0 o 1 supervisor directo.
- Un supervisor puede tener 0 o N empleados a cargo.
- Admin ve toda la organización (todas las sucursales).
- Config por sucursal (opcional): horario, lugar pueden ser por sucursal. Por defecto, config es a nivel org.

--- 2.3 Tablas ---
organizations (id, name, ...)
branches (id, org_id, name, address, ...)  -- sucursales
employees (id, org_id, branch_id, supervisor_id, status, fecha_ingreso, fecha_egreso, ...)
places (id, org_id, branch_id, nombre, direccion, lat, long, radio_m, dias, ...)

--- 2.4 Flexibilidad ---
CFG-ORG-001: estructura_sucursales_habilitada (boolean, default false)
- Si false: branch_id nullable, todos en raíz.
- Si true: se pueden crear sucursales. Empleados y lugares se asignan a sucursal.
- Un empleado puede cambiar de sucursal (histórico en audit).

--- 2.5 Reportes por sucursal ---
Filtro exportación: Todas | Sucursal X | Varias sucursales.
Supervisor de sucursal A solo ve reportes de sucursal A.
Admin ve por sucursal o global.

7. CASOS LÍMITE (CL-001 a CL-043)
================================================================================

CL-001: Fichaje fuera zona → Botón deshabilitado. "Estás fuera de tu zona..." Log + notif Supervisor.
CL-002: Lugar no asignado para ese día → Permitir si remoto y en cualquier lugar org; rechazar si config estricta.
CL-003: GPS deshabilitado → "Activa la ubicación para poder fichar."
CL-004: GPS impreciso (>50m) → "Esperá un momento, mejorando precisión..." timeout 30s.
CL-005: Zona límite (dentro tolerancia) → PERMITIR.
CL-006: Dos entradas seguidas → Backend rechaza 400. Debounce 5s frontend.
CL-007: Descanso <12h → Bloquear o advertir según CFG-011. Mensaje con horas restantes.
CL-008: Descanso exacto 12h → PERMITIR.
CL-009: Turno rotativo excepción → Si CFG-013 y descanso_especial config: permitir según convenio.
CL-010: Fichaje offline → Guardar local cifrado. Sync al reconectar. Timestamps dispositivo + servidor.
CL-011: Offline prolongado → Sync orden cronológico. Conflictos: más antiguo gana, otro "conflicto".
CL-012: Offline con CFG-009=false → "Sin conexión. Conectate para fichar."
CL-013: Dispositivo nuevo (límite) → "Alcanzaste límite. Revocá uno o solicitá a Admin."
CL-014: Primer login dispositivo nuevo → Email notificación.
CL-015: Sesión expirada → Redirigir login. "Tu sesión expiró por inactividad."
CL-016: Licencia sin adjunto (obligatorio) → "Para enfermedad/accidente debés adjuntar certificado."
CL-017: Adjunto formato no permitido → "Solo PDF, JPG, PNG hasta 5 MB."
CL-018: Solapamiento licencias → "Las fechas se superponen con licencia del X al Y."
CL-019: Aprobar sin ver adjunto → Permitir. Advertencia "¿Revisaste el certificado?" checkbox.
CL-020: Admin editar fichaje → NO directo. Solo corrección con justificación, nuevo registro, original marcado.
CL-021: Borrar fichaje → NUNCA. Solo soft-delete Admin con motivo.
CL-022: Manipulación hash → Servidor calcula hash. Cliente no puede. Cadena rompe si modificación directa.
CL-023: Dispositivo poca memoria → Modo low-end automático.
CL-024: Batería baja → Reducir geofencing. No bloquear fichaje manual.
CL-025: Doble click Fichar → Debounce 300ms. Idempotency key. Segundo request no duplica.
CL-026: Import email duplicado → Error fila. Continuar resto. Resumen final.
CL-027: Import CUIL inválido → Rechazar fila.
CL-028: Lugar coordenadas inválidas → Validar lat[-90,90], long[-180,180]. Geocoding si falla.
CL-029: Licencia futuro >1 año → "Las fechas no pueden superar 1 año desde hoy."
CL-030: Timesheet horas > fichadas → "Las horas registradas superan las trabajadas."
CL-031: Empleado a ruta Admin → 403. "No tenés permiso."
CL-032: Auditor intenta modificar → 403.
CL-033: Rate limit login → 5 intentos/5min → bloqueo 15min. Mensaje genérico.
CL-034: JWT expirado → 401. Frontend redirige login.
CL-035: Webhook falla → Retry 3x backoff. No bloquear flujo. Log.
CL-036: Export >10k filas → Background. Email con link 24h. Límite 500k.
CL-037: Empleado despedido intenta fichar → 403. "Tu cuenta fue desactivada."
CL-038: Empleado despedido intenta login → Auth falla. Mensaje genérico.
CL-039: Eliminar sucursal con empleados → Rechazar. "Reasignalos antes." O soft-delete.
CL-040: Supervisor despedido con equipo → Requerir reasignación empleados antes de confirmar.
CL-041: Filtro fechas invertidas → "La fecha desde debe ser anterior a la hasta."
CL-042: Export sin empleados seleccionados → "Seleccioná al menos un empleado o 'Todos'."
CL-043: Último Admin cambiar rol → Rechazar. "Debe haber al menos un administrador."

8. EMPLEADO DESPEDIDO (OFFBOARDING)
================================================================================

Acción "Dar de baja": fecha_egreso obligatorio, motivo opcional.
Confirmación: "¿Desactivar cuenta de [nombre]? No podrá acceder. Datos se conservan."
Al confirmar: status=despedido, invalidar TODAS sesiones, revocar dispositivos.
No eliminar datos históricos. Log: empleado_despedido.

Restricciones: No login, no fichar, no licencias. No en listas activos. Sí en reportes históricos.
Admin puede filtrar "ver despedidos". Legal_auditor ve todo para peritaje.

Reingreso: Opción "Reactivar" → status=activo, fecha_reingreso. Mantener historial.

10. SEGURIDAD
================================================================================

Requisitos legales para juicios: ver VALIDEZ-PROBATORIA-JUICIOS-ARGENTINA.txt.

Cadena hashes: SHA-256, timestamp servidor, hash_registro, hash_anterior. No UPDATE fichajes. Corrección = nuevo registro.
Auth: bcrypt/Argon2id. 2FA obligatorio Admin. JWT exp corto + refresh. Rate limit 5 intentos/5min.
RLS: org_id, branch_id en políticas. Auditor solo SELECT audit_logs.
Audit logs: Solo INSERT. Sin UPDATE/DELETE. Retención CFG-037.
Cifrado: HTTPS obligatorio. DB cifrada. Offline local cifrado.
Headers: X-Content-Type-Options, X-Frame-Options, CSP, Referrer-Policy.
Idempotency: Fichaje con idempotency key.
Checklist pre-deploy: env, HTTPS, RLS, audit, rate limit, 2FA Admin, backups, CORS.

Import lugares: nombre, direccion, lat, long, radio_m, dias. Geocoding si sin coords.

Webhooks: HTTPS, evento, payload JSON, X-Signature HMAC. Eventos: fichaje.creado, licencia.*, empleado.*, alerta.generada. Retry 3x.
ARCA: Futuro. Altas/bajas. No bloquear si no disponible.
API integración: GET /integrations/fichajes. API Key. Rate 100 req/min.
Límites: Import empleados 1000, lugares 200, reporte 500k, adjunto 5MB.

fichar-bug-audit: Auditar bug, hipótesis causa, archivos afectados, pasos repro, sugerencia fix.
fichar-bug-fix: Fix mínimo, test que repro y verifica. Usar tras hipótesis confirmada.

Errores comunes:
| Error | Causa | Acción |
|-------|-------|--------|
| Fichaje duplicado | Doble click | Idempotency key |
| "Fuera zona" erróneo | GPS impreciso | Aumentar tolerancia |
| Session expired | Timeout | Guardar draft, renovar |
| Export vacío | Filtros excluyen | Validar, mensaje claro |
| 403 inesperado | RLS/rol | Log, verificar org_id |
