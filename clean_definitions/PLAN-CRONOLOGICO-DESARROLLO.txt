# fichAR — Plan Cronológico de Desarrollo

Plan preciso con puntos de test obligatorios. No avanzar de fase con tests rojos.

================================================================================
CONVENCIONES
================================================================================
[TEST] = Checkpoint obligatorio de tests
[REVIEW] = Revisión de código/arquitectura
Duración en semanas laborales (5 días)
Cada fase termina con: bun test && flutter test && bun run lint && flutter analyze
