# fichAR Skill Planning

## When to Use

- Non-trivial feature or refactor
- Multi-file changes
- Before implementing complex logic

## Output

Create `docs/plans/<feature>.md` in **English** with:

1. **Goal**: What we're building
2. **Scope**: In scope / out of scope
3. **Approach**: High-level steps
4. **Files affected**: Estimated list
5. **Dependencies**: Definitions, configs
6. **Risks**: Edge cases, considerations

## Workflow

1. Produce plan
2. Review against definiciones/
3. Implement after approval
4. Update plan with actual changes if diverged
